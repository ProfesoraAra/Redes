Modelos OSI y TCP/IP

Modelo de Referencia OSI

Publicado en 1984 por la ISO, el modelo Open Systems Interconnection es un marco conceptual que divide las funciones de red en 7 capas. Aunque no es una arquitectura comercial, es la herramienta pedagógica por excelencia para el diseño modular y el diagnóstico de fallas.

Estructura por Niveles

El modelo garantiza que el hardware y el software de diferentes fabricantes puedan interoperar de forma predecible.

7. Aplicación
6. Presentación
5. Sesión
4. Transporte
3. Red
2. Enlace de Datos
1. Física

Características Clave

  • Independencia: Cambios en una capa no afectan el funcionamiento de las demás.
  • Modularidad: Facilita el desarrollo de protocolos específicos para tareas concretas.
  • Jerarquía: Cada nivel presta servicios al superior y consume del inferior.

Arquitectura TCP/IP

Aplicación HTTP, FTP, DNS, SMTP
Transporte TCP, UDP (Mapeo directo con OSI)
Internet IP, ICMP, ARP (Ruteo Universal)
Acceso a la Red Ethernet, Wi-Fi, Medios Físicos

El modelo TCP/IP (Transmission Control Protocol / Internet Protocol) es el pilar real de las redes modernas. A diferencia de OSI, es un modelo pragmático que consolida las funciones para maximizar la velocidad y la eficiencia en la transmisión de datos.

Filosofía de Diseño

Fue diseñado para sobrevivir a fallos masivos en la infraestructura de red, permitiendo que los datos encuentren rutas alternativas automáticamente.

Eficiencia Menos capas, menos retardo.
Pragmatismo Enfoque en protocolos reales.
