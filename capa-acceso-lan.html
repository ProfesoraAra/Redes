Seguridad y hardening en la capa de acceso LAN

Configuración de contraseña

La primera línea de defensa de cualquier dispositivo de red (switches o routers) reside en el endurecimiento de sus accesos lógicos de administración. El sistema operativo Cisco IOS permite configurar niveles independientes de acceso para proteger el plano de control, restringir la terminal interactiva (CLI) y evitar intrusiones no autorizadas por fuerza bruta.

Directivas de Acceso y Hardening

Clave de Privilegio (Secret) Asegura el paso al modo de ejecución privilegiado. El uso de enable secret aplica cifrado seguro unidireccional (SHA-256 o MD5), suplantando al vulnerable enable password.
Cifrado Tipo 7 El comando service password-encryption aplica una máscara reversible a todas las claves del sistema que se visualizan en texto plano al ejecutar un volcado de la configuración activa.
Longitud Mínima Forzar claves robustas mediante el comando global security passwords min-length [n] evita la configuración de claves débiles o por defecto.
Bloqueo de Fuerza Bruta La sentencia de configuración login block-for [seg] attempts [n] within [seg] inhabilita temporalmente los intentos de sesión sospechosos por un periodo de tiempo determinado.

Resistencia contra Fuerza Bruta

Eficacia de mitigación (Tasa de bloqueo) al combinar directivas de complejidad de contraseñas.

Herramientas de seguridad

Cifrado, autenticación centralizada e integridad de red

SSH vs Telnet

A diferencia del protocolo Telnet, que transmite credenciales y comandos en texto plano, SSH implementa criptografía RSA asimétrica para asegurar la confidencialidad absoluta de la sesión de administración a través del puerto 22.

Esquema AAA

Proporciona un marco de control de acceso centralizado. Autenticación (valida quién eres), Autorización (determina qué privilegios posees y qué comandos puedes aplicar) y Auditoría (mantiene registros de todas las acciones).

Inspección DHCP/ARP

Medidas de protección de Capa 2 contra servidores DHCP piratas (DHCP Snooping) y envenenamiento de tablas de traducción de direcciones (Dynamic ARP Inspection), mitigando ataques Man-In-The-Middle.

Comandos Esenciales para Configurar SSH

Secuencia recomendada para la habilitación de una consola remota cifrada sobre IOS.

Switch(config)# hostname SW-PRINCIPAL
SW-PRINCIPAL(config)# ip domain-name unrc.edu.mx
SW-PRINCIPAL(config)# crypto key generate rsa (Sugerido: 1024 o 2048)
SW-PRINCIPAL(config)# username admin privilege 15 secret CLAVE_SEGURA
SW-PRINCIPAL(config)# line vty 0 15
SW-PRINCIPAL(config-line)# transport input ssh
SW-PRINCIPAL(config-line)# login local

Configuración de la seguridad de los puertos

La seguridad de puertos (Port Security) limita de manera activa la cantidad de direcciones MAC permitidas en una interfaz física de acceso del switch. Esta directiva neutraliza ataques devastadores de Capa 2 como el desbordamiento de tablas de direcciones MAC (MAC Flooding) y el uso no autorizado de interfaces físicas expuestas en áreas comunes.

Modos de Aprendizaje MAC

Tipos de registro de direcciones en interfaces con Port-Security.

Estático: Definido manualmente por comandos en la CLI. Almacenado de forma permanente en la configuración del dispositivo.
Dinámico: Registrado automáticamente en la tabla MAC activa del switch. Se pierde ante un reinicio del hardware.
Sticky (Adhesivo): El dispositivo aprende la dirección de forma dinámica y la añade de inmediato a la configuración activa (running-config).

Modos de Violación de Puerto

Comportamiento operativo detallado para cada modo ante la detección de una dirección MAC intrusa.

Acción Operativa Protect Restrict Shutdown
Descarta tráfico intruso ✅ Sí ✅ Sí ✅ Sí
Genera registro Syslog ❌ No ✅ Sí ✅ Sí
Incrementa contador de alertas ❌ No ✅ Sí ✅ Sí
Deshabilita el puerto físico ❌ No ❌ No ✅ Sí (err-disabled)

Comandos para Implementar Port Security

Implementación de un puerto seguro limitando accesos con aprendizaje dinámico persistente.

SW-PRINCIPAL(config)# interface FastEthernet 0/1
SW-PRINCIPAL(config-if)# switchport mode access
SW-PRINCIPAL(config-if)# switchport port-security
SW-PRINCIPAL(config-if)# switchport port-security maximum 2
SW-PRINCIPAL(config-if)# switchport port-security mac-address sticky
SW-PRINCIPAL(config-if)# switchport port-security violation restrict
